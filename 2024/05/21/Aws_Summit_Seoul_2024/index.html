    컨퍼런스는

    [10주년을 기념하며 AWS 세션이 진행되는 3층 오디토리움]

    어김없이 올해도 AWS Summit Seoul을 개최할 시즌이 돌아왔다. 올해는 2주 정도 늦은 5월 16일(목) ~ 5월 17(금)로 이틀동안 진행될 예정인데, 평소와는 다르게 10주년이라고 한다. (AWS 행사가 2014년에 처음 개최되었다는걸 처음 알았다.) 작년에 이어 올해도 AI가 주요 키워드로 행사가 진행된다고 하는데 기대되는 부분이 많았다.

    세션 선택하기

    [1일차 세션]

    [2일차 세션]

    작년 Summit에서 ML에 대한 이론과 활용하는 세션들과 튜닝해서 모델을 생성하는 세션을 들었었는데, 실제 활용하기에는 더 많은 공부가 필요했다. 계속해서 틈틈이 공부를 했지만 실습과 별개로 실제 서비스에 활용할 기회는 많지 않았고, 전체적인 아키텍처를 구상해보았을 때, 여전히 궁금한 부분이 많았다.

    그리고 올해 세션을 살펴보니 AI와 MLOps에 대한 세션들이 주를 이루고 있었다. 작년은 모델을 만들고 고도화를 하는데 집중했었으니, 올해는 만든 모델을 활용하는데 필요한 데이터 파이프라인, RAG, 보안 등에 대한 내용을 다루는 것 같았다.

    평소 생산성 향상에 굉장히 관심이 많았던 나는 생산성 향상을 위해 AI 서비스를 활용한 세션들을 듣고 싶었고, MLOps, GenAI, RAG 키워드를 중점으로 세션을 선택하게 되었다.

    컨퍼런스 참여

    올해는 이틀 모두 참석했고 오전부터 참관하였다. 휴가를 사용하고 방문해야 했던 작년, 2일날 잠깐 참석했던 것과 늦게 등록해서 종이 명찰 + 민무늬 카드 형태의 네임택을 받았던 것이 아쉬웠기 때문이다. 그래서 올해는 EXPO 참관과 함께 여러 세션을 최대한 듣고 카드 형태의 네임택을 받고자 하는 목표를 세웠다.

    [올해 등록이 진행중인 3층 등록 부스]

    1일째 아침, 나는 10시 40분쯤 도착하여 등록 부스로 향했다. 여전히 지하철부터 사람들이 많았고, ‘설마 AWS 참석하는 사람들인가’ 싶었던 인파는 그대로 등록 부스까지 이어졌다…ㅎ 올해 등록은 3층에서 진행되었는데, 더욱 화려해졌다. (작년 등록을 진행했던 홀에서 다른 행사가 진행중이던데 이거 때문인가?)

    [올해는 카드 형태의 네임택을 받았다!]

    등록을 마치고 네임택을 수령했을 때, 아직 기조연설이 진행중이었다. 기조연설은 중앙 기둥에 4개의 스크린을 붙인 형태로 구성되어 사람들이 둘러 앉아 있었고, AWS Q Developer 서비스에 대해 번역되어 나오는 내용을 듣고 있었다.

    [기조연설 중인 등록 부스 뒤쪽 홀]

    EXPO부터 참관하기

    [등록 부스에서 바로 이어져 있는 EXPO 연결 통로]

    등록을 마친 뒤, 나는 EXPO를 먼저 참관하게 되었다. EXPO 참관에 대해 의외로 많은 고민이 되었는데, 작년 EXPO 참관은 나에게 큰 아쉬움으로 남아 있기도 했고 세션들을 집중해서 듣는게 나을거 같았기 때문이다. 작년에는 점심 지나고 와서 시간이 부족했던 이유로 10분 정도 구경만 했었다. 올해는 세션이나 EXPO나 둘 다 집중해서 참관하고 싶었다.

    [EXPO 내부]

    EXPO로 들어갔을 때, 생각보다 사람이 많지 않은거 같았다. 일찍 와서 그런가… 싶었지만 전혀 아니었다.^^

    [AMD에서 설문 참여하고 받은 첫 기념품]

    AMD 부스 방문했다가 둘러보니 여러 부스에 많은 인파가 있었다.

    [많은 인파가 있는 EXPO]

    [AWS에서 준비해놓은 포토존]

    [AWS 부스에서 퀴즈를 맞히고 받은 코인!]

    [코인으로 뽑기를 해서 받은 친환경 씨앗 연필]

    [심으면 실제로 토마토가 난다던데…? 한 번 열심히 키워봐야지.]

    [AWS 전문가들의 자유로운 Q&A 부스]

    이번 EXPO의 다양한 기업의 부스를 보면서 나는 한가지 기준을 가지고 보았다. 그 관점은 어려운 문제를 풀고 있는 기업들의 시행착오 결과가 어떤 UI/UX 그리고 기능으로 제품에 녹여졌는지였다. 이걸 기준으로 삼았던 이유는 이러한 경험이 나중에 나에게 다양한 관점을 가지게 해줄 것이라고 생각해서이다.

    기업들은 각자 가진 자본, 기술, 문화가 규모나 밀도 면에서 모두 다르다고 생각한다. 그리고 이런 환경 속에서 비슷하거나 다른 문제들을 각자 자기만의 방식으로 풀어내고 있을 것이다.

    각 기업들의 제품을 보면서 생각하지 못했던 접근 방식을 생각하게 되고, 내가 가진 문제를 해결할 수도 있는 인사이트를 얻기에, EXPO는 나에게 좋은 기회였다.

    [여러 부스 구경과 함께 이벤트를 참여하여 받은 기념품들]

    이렇게 여러 부스를 돌면서, 새로운 기업들을 알게 되었고 AI, 데이터 관련하여 기업마다 어떤 포인트에 집중하는지 제품을 통해 엿볼 수 있어 좋았다. 내가 관람했던 기준에서 한단계 더 나아가, 기업들의 제품에서 어떤 문제를 부딪혔길래 현재의 UI/UX가 나왔을지 역으로 추론도 시도해보았다. (아직 경험이 많지 않아서 그런지 확신이 들거나 한 번에 보이는 것은 없었다. 좀 더 경험이 쌓이고 다시 참석할 때는 더 많은 것이 보이지 않을까 기대가 된다.)

    세션에 집중하기 위해 1시간 반 가량을 사용하여 먼저 관람했던 EXPO는 기대했던 부분들을 충분히 채워주어 만족스러웠다.

    여러가지 세션을 들으며

    [세션 들으러 3층 E홀 들어가는 길]

    작년에는 옛날 폰을 사용해서 배터리 호흡기(보조배터리나 콘센트)가 필수여서, 듣는데 힘든 부분 중 하나였다… 하지만 마.침.내! 최신 폰으로 바꾸고 배터리로부터 자유로워질 수 있었다…! (비록 하루종일 카메라를 써야 해서 살짝의 보조배터리는 필요했지만) 그래서 작년과는 다르게 준비된 새 폰을 가지고 세션의 내용을 더 잘 담아내기 위해 강연장의 좌석을 고르는데 더 집중을 하였다.

    [4층 401호에서 세션을 듣기 위해 앉아있는 자리]

    [세션을 듣기 위해 강연장에 입장한 많은 사람들]

    작년에는 AI/ML 관련해서 모델을 학습하는 기법에 대한 얘기들을 많이 들었다면, 올해는 많아진 모델들을 어떻게 서빙하고, 데이터를 어떻게 관리하며, 이를 기능으로 어떻게 제공할 수 있을지에 대한 MLOps쪽 내용과 ML 활용에 대한 내용들이 많았던거 같다.

    이 과정에서 AWS의 다양한 서비스들과 파트너 업체의 다양한 솔루션들을 보게 되었는데, 자주 접한 AI와 관련된 서비스를 정리하면 다음과 같다.

    • 생성형 AI
      • AWS Bedrock
        • 생성형 AI를 활용할 수 있도록 서비스 제공 (모델 임베딩, 파운데이션 모델 활용)
        • 요구사항 단계에서 사용
      • AWS Knowledge Bases (for Amazon Bedrock)
        • Bedrock에서 활용할 수 있도록 자료를 임베딩하고 벡터 데이터베이스 형태로 관리해주는 서비스
    • 머신러닝
      • AWS SageMaker
        • 머신러닝에 대한 기능을 활용할 수 있도록 제공하는 서비스 (Model 생성, 학습 등)
    • 개발자 경험 관련
      • AWS DevOps Guru
        • 애플리케이션 운영 문제를 자동으로 감지하고 해결책을 제안하는 ML 기반 운영 도우미 서비스
      • AWS CodeGuru
        • 코드 품질을 자동으로 리뷰하고 성능 개선을 위한 제안을 제공하는 ML 기반 코드 분석 서비스
      • AWS CodeCatalyst
        • 클라우드 기반 통합 개발 환경으로, 프로젝트 관리, 코드 빌드 및 배포 파이프라인을 간소화하여 개발 효율성을 높이는 서비스
      • AWS Q Developer
        • 기존 코드와 사용자가 입력한 자연어를 기반으로 코드를 제안하고 생성하는 강력한 생성형 AI 서비스

    앞에서 소개한 서비스 외에도 세션에서 보안을 위한 MLOps나, Imply의 druid 솔루션처럼 다양한 내용들을 세션들에서 접할 수 있었다. 또 여러 기업들이 어떤 문제를 가지고 있었고, 이를 어떻게 풀어나갔는지 들을 수 있어 좋았다. (자세한 내용은 별도의 포스트로 정리할 예정이다)

    마치며

    [AWS Summit Seoul 2024 굿즈로 제공된 티셔츠]

    올해는 텀블러 대신 티셔츠를 받았다. 난 개인적으로 텀블러 같은 게 더 좋았는데, 다행히 화려하지 않고 무난했다. (내년엔 옷 말고 다른 거로…!)

    작년 컨퍼런스 행사에 참여한 후 아쉬운 부분들을 정리하고, 올해는 나름의 기준으로 목적을 가지고 참석해서 그런지 EXPO, 세션, 개인적으로 느낀 부분 등 전반적으로 만족스러운 경험이 되었다. 틈틈이 AI쪽 공부를 했던 게 도움이 되었는지, 작년과 다르게 세션의 내용이 귀에 더 들어오고 이해되기도 했고, EXPO를 참관하여 더욱 컨퍼런스를 알차게 참석하였다.

    그리고 의외로 개인적인 부분에서 크게 얻는 것이 있었다. 작년 연말부터 올해까지 번아웃을 겪어서 자극이 필요하였는데, AWS Summit은 여러모로 좋은 자극을 준 행사였다. 작년과 마찬가지로 각자가 가진 문제를 어떤 방식으로 멋지게 풀어냈는지, 밀도 있고 깔끔하게 전달하는 모습은 정말 멋졌다. 또 각 세션의 발표는 매력적인 조직이 많고, 그들과 함께 문제를 해결하면서 성장을 하고 싶다는 생각이 들게 하였는데, 이런 자극은 얼른 여러 문제를 접하고 해결하면서 성장하고 싶다는 동기를 유발하였다.

    올해 Summit이 끝나고 다음 시즌이 시작되는 느낌이 들었다. 아마 곧 새로운 환경에서 새로운 사람들과 함께 문제를 해결할 것이기 때문이지 않을까 싶다. 이전 회사를 그만둔 지 2주 정도 되었고, 2주 동안 개인적인 일들을 정리하며 새로운 회사로 출근하기 위한 준비를 하였다. 이번 컨퍼런스를 통해 느낀 설렘과 다시 충전한 에너지를 가지고, 곧 시작될 새로운 환경에 잘 적응해 나가야겠다.
