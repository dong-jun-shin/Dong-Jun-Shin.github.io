    들어가며

    지난 포스트에서는 순열과 조합이 무엇인지, 어떤 원리로 경우의 수를 얻을 수 있는지에 대해 이해해보았다. 이번에는 이해한 원리를 바탕으로 순열과 조합을 구현해보자.

    순열과 조합

    이 포스트에서는 구현 위주로 작성할 예정이기 때문에, 코드가 많이 나올 예정이다. 이론적인 내용은 ‘순열과 조합, 어떤 원리로 경우의 수를 계산할 수 있는걸까?’ 를 참고하길 바란다.

    이 포스트의 목표는 다음과 같다.

    1. 순열과 조합의 공식을 바탕으로 함수를 구현해보며 공식의 이해를 높인다.
    2. 구현된 함수를 실제 사용할 수 있도록 로직을 개선한다.

    이 목표들을 통해 순열과 조합 수를 구하는 것에서 나아가, 각 케이스를 직접 구하는 함수를 작성하며 순열과 조합의 이해를 마무리 하려 한다.

    구현

    앞서 공식을 다시 확인하면 다음과 같다.

    • 순열: $nPr = n! / (n-r)!$
    • 조합: $nCr = n! / r!(n-r)!$

    구현을 위해 이 공식을 그대로 사용하면 경우의 수가 몇 가지인지 구할 수 있을 것이다. 하지만 계산된 경우의 수에 어떤 경우(이하 case)들이 있는지는 알 수 없다.

    case를 구하면 자연스럽게 경우의 수까지 구할 수 있을테니, 공식을 통해 case를 구하는 함수를 작성해보자.

    순열

    먼저 case를 구하는 함수를 작성하기 위해 함수 스펙과 공식을 의사 코드로 정리하여 단계별로 명확하게 해보자.

    Function Signature

    함수에 대해 input, output, function name을 정리해보면 다음과 같다.

    • input: 서로 다른 전체 원소 n, 선택하는 원소 갯수 r
    • output: 서로 다른 전체 원소 중에서 r개를 택하여 일렬로 나열하는 경우의 수
    • function name: permutation

    Pseudo Code

    다음은 수행할 단계이다.

    1. 서로 다른 전체 원소 n, 선택하는 원소 갯수 r을 입력받는다.
    2. 전체 원소 n에 대해 모든 원소를 나열하는 경우 $n!$을 구한다.
    3. 선택하지 않은 원소를 제외하고 중복된 case를 제거하여 $/(n-r)!$을 적용한다.
    4. 3번의 결과를 반환한다.

    Code

    이제 의사 코드를 바탕으로 함수를 구현해보자. (순열과 조합 구현에 집중하기 위해, 같은 원소인지 판단하는 함수는 생략한다.)

    단순 구현

    우선, 문자열 ‘a, b’로 이루어진 배열에서 2개를 선택하는 경우의 수를 구한다고 가정하고 구현해보겠다.

    function isEqualElement(a: any[], b: any[]): boolean {
  /** 배열 형태로 이루어진 원소들이 같은지 확인하는 함수 */
}

function permutations(elements: any[], select: number): any[][] {
  const allCases: string[][] = [];

  // NOTE: `n!`에 해당하는 부분
  for (let i = 0; i < elements.length; i++) {
    for (let j = 0; j < elements.length; j++) {
      if (i === j) continue;
      const first = elements[i];
      const second = elements[j];

      allCases.push([first, second]);
    }
  }

  // NOTE: 선택하지 않은 요소를 제외, `/ (n - select)!`에 해당하는 부분
  const permutationCases: string[][] = [];

  for (const oneCase of allCases) {
    // NOTE: 1개를 선택하는 경우 전체를 선택한 케이스에서 1개를 제외, [['a', 'b'] => ['a']]
    const pickedCase = oneCase.slice(0, select);

    if (permutationCases.some((value) => isEqualElement(pickedCase, value))) continue;
    permutationCases.push(pickedCase);
  }
  return permutationCases;
}

permutations(["a", "b"], 2); // [['a', 'b'], ['b', 'a']]

    위 코드는 3개의 문자열 중 2개를 선택하는 경우를 예시로 의사코드를 구현한 것이다. 이 코드는 이해를 위해 직관적으로 작성한 코드이기 떄문에, 선택하는 갯수에 따라 for문이 중첩되는 문제가 있다. 이를 해결하기 위해 재귀를 활용한 구현을 시도해보자.

    재귀 구현

    이번엔 전체 갯수 n개 중, r개를 선택하는 경우를 고려하여 재귀를 활용하는 형태로 코드를 수정할 것이다.

    function isEqualElement(a: any[], b: any[]): boolean {
  /** 배열 형태로 이루어진 원소들이 같은지 확인하는 함수 */
}

// NOTE: 모든 원소를 선택한 case를 생성 (재귀 구현)
function retrieveAllCases(elements: any[]): any[][] {
  if (elements.length === 1) return [elements];

  const allCases: any[][] = [];

  for (let i = 0; i < elements.length; i++) {
    const rest = retrieveAllCases(elements.filter((_, idx) => idx !== i));

    allCases.push(...rest.map((el) => [elements[i], ...el]));
  }

  return allCases;
}

function permutations(elements: any[], select: number): any[][] {
  // NOTE: `n!`에 해당하는 부분
  const allCases = retrieveAllCases(elements);

  // NOTE: 선택하지 않은 요소를 제외, `/ (n - select)!`에 해당하는 부분
  const permutationCases: string[][] = [];

  for (const oneCase of allCases) {
    const pickedCase = oneCase.slice(0, select);

    if (permutationCases.some((value) => isEqualElement(pickedCase, value))) continue;
    permutationCases.push(pickedCase);
  }
  return permutationCases;
}

permutations(["a", "b", "c"], 2); // [['a', 'b'], ['a', 'c'], ['b', 'c']]

    단순 구현한 코드에서 선택하는 개수만큼 for문이 중첩되었던 문제를 해결하기 위해, ‘n!’에 해당하는 for문 코드를 함수로 분리하고 재귀로 구현하였다. 이제 구현한 함수를 활용하여 순열을 구할 수 있게 되었다.

    하지만 이 코드는 공식 이해를 위해 작성한 코드이기 때문에, 실제 사용하기에는 불필요한 동작들이 포함되어 있다. 마지막으로 이를 개선하는 코드를 작성해보자.

    최적화

    공식을 기반으로 작성한 위 코드는 이해하기에는 좋았으나, 더 간소화할 수 있는 동작들이 있다.

    우리는 전체 n개 중 r개를 선택할 수 있는 경우를 알고 싶은 것이기 떄문에 공식에 있던 것처럼 전체를 구할 필요는 없다. (공식의 경우 경우의 수를 구하는 것이고, 작성중인 함수는 경우를 구하는 것이기 떄문에 차이가 있다.)

    다시 말해, 전체를 구하는 것이 아닌, 선택하는 갯수만큼만 구하면 되기 때문에, 이를 고려하여 코드를 수정한다면 전체 case를 구하는 코드, 원소가 같은지 체크하는 함수, 중복된 case를 제거하는 코드를 제거할 수 있다.

    이를 반영하여 코드를 수정하면 다음과 같다.

    function permutations(elements: any[], select: number): any[][] {
  if (select === 1) return elements.map((el) => [el]); // [[firstPick], [secondPick], ...]
  const permutationCases: any[][] = [];

  // NOTE: n개를 순회하며 재귀함수를 호출, 재귀 함수 depth는 곧 원소 선택을 의미한다.
  for (let i = 0; i < elements.length; i++) {
    const restPickCases = permutations(
      elements.filter((_, idx) => idx !== i),
      select - 1
    );
    const permutationCase = restPickCases.map((restPickCase) => [elements[i], ...restPickCase]); // [[firstPick, secondPick, ...], ... ]

    permutationCases.push(...permutationCase);
  }

  return permutationCases;
}

permutations(["a", "b", "c"], 2); // [['a', 'b'], ['a', 'c'], ['b', 'a'], ['b', 'c'], ['c', 'a'], ['c', 'b']]

    전체 case를 구하는 코드와 중복된 case를 제거하는 코드를 제거하고, 재귀 함수를 활용하여 선택하는 갯수만큼만 구하는 코드로 수정하였다. 지금까지 과정을 이해했다면 순열 공식과 구현은 끝이다.

    조합의 구현이 남아있기는 하지만, 조합은 공식에서 보았듯이 순열을 활용하면 간단하다. 마지막으로 조합을 구현하고 마무리하자.

    조합

    Function Signature

    함수에 대해 input, output, function name을 정리해보면 다음과 같다.

    • input: 서로 다른 전체 원소 n, 선택하는 원소 갯수 r
    • output: 서로 다른 전체 원소 중에서 r개를 택하는 경우의 수
    • function name: combination

    순열과 매우 흡사하다.

    Pseudo Code

    다음은 수행할 단계이다.

    1. 서로 다른 전체 원소 n, 선택하는 원소 갯수 r을 입력받는다.
    2. 전체 원소 n에 대해 모든 원소를 나열하는 경우 $n!$을 구한다.
    3. 선택하지 않은 원소를 제외하고 중복된 case를 제거하여 $/(n-r)!$을 적용한다.
    4. 선택한 원소들을 나열할 수 있는 case를 중복으로 여기고 제거하여 $/r!$을 적용한다.
    5. 4번의 결과를 반환한다.

    순열에서 4번만 추가되었다. 이 부분을 활용하여 순열 함수를 재활용해서 간단하게 구현해볼 수 있다.

    Code

    이제 의사 코드를 바탕으로 함수를 구현해보자. (순열은 앞에서 설명하였기 때문에, 순열 부분에 해당하는 코드는 함수로 대체하고 생략한다.) (조합 구현에 집중하기 위해, 같은 원소인지 판단하는 함수는 생략한다.)

    단순 구현

    1, 2, 3번 부분은 순열과 동일하다. 4번 부분만 추가로 구현해보겠다.

    function isEqualElement(a: any[], b: any[]): boolean {
  /** 배열 형태로 이루어진 원소를 순서 상관없이 같은지 확인하는 함수 */
}

function permutations(elements: any[], select: number): any[][] {
  /** n개 중 r개를 선택하는 경우를 나열한 배열을 반환하는 함수 */
}

function combinations(elements: any[], select: number) {
  if (select === 1) return elements.map((el) => [el]);

  // NOTE: `n! / (n - select)!`에 해당하는 부분
  const permutationCases = permutations(elements, select);

  // NOTE: 순열의 경우의 수 중 중복되는 경우를 제외, `/ select!`에 해당하는 부분
  const combinationCases: any[][] = [];

  for (const oneCase of permutationCases) {
    if (combinationCases.some((combinationCase) => isEqualElement(combinationCase, oneCase))) continue;
    combinationCases.push(oneCase);
  }

  return combinationCases;
}

combinations(["a", "b", "c"], 2); // [['a', 'b'], ['a', 'c'], ['b', 'c']]

    for문 하나와 순서 상관없이 같은 원소로 구성되었는지 확인하는 함수만 추가하여 구현하였다. 이제 조합 공식과 구현의 이해도 끝났다.

    테스트

    이제 작성한 함수를 테스트해보자. 테스트는 5가지 유형의 input으로 함수를 호출하여 결과를 확인하였다.

    // Case 1 (문자로 이루어진 3개 원소 중 2개를 선택하는 경우)
// 6가지, [['a', 'b'], ['b', 'a'], ['a', 'c'], ['c', 'a'], ['b', 'c'], ['c', 'b']]
console.log(permutations(["a", "b", "c"], 2));
// 3가지, [['a', 'b'], ['a', 'c'], ['b', 'c']]
console.log(combinations(["a", "b", "c"], 2));

// Case 2 (숫자 배열로 이루어진 3개 원소 중 2개를 선택하는 경우)
// 6가지, [[[1, 2], [3, 4]], [[1, 2], [5, 6]], [[3, 4], [1, 2]], [[3, 4], [5, 6]], [[5, 6], [1, 2]], [[5, 6], [3, 4]]]
console.log(
  permutations(
    [
      [1, 2],
      [3, 4],
      [5, 6],
    ],
    2
  )
);
// 3가지, [[[1, 2], [3, 4]], [[1, 2], [5, 6]], [[3, 4], [5, 6]]]
console.log(
  combinations(
    [
      [1, 2],
      [3, 4],
      [5, 6],
    ],
    2
  )
);

// Case 3 (객체로 이루어진 3개 원소 중 2개를 선택하는 경우)
// 6가지, [[{ a: 1 }, { b: 2 }], [{ a: 1 }, { c: 3 }], [{ b: 2 }, { a: 1 }], [{ b: 2 }, { c: 3 }], [{ c: 3 }, { a: 1 }], [{ c: 3 }, { b: 2 }]]
console.log(permutations([{ a: 1 }, { b: 2 }, { c: 3 }], 2));
// 3가지, [[{ a: 1 }, { b: 2 }], [{ a: 1 }, { c: 3 }], [{ b: 2 }, { c: 3 }]]
console.log(combinations([{ a: 1 }, { b: 2 }, { c: 3 }], 2));

// Case 4 (문자로 이루어진 4개 원소 중 2개를 선택하는 경우)
// 12가지, [['1', '2'], ['1', '3'], ['1', '4'], ['2', '1'], ['2', '3'], ['2', '4'], ['3', '1'], ['3', '2'], ['3', '4'], ['4', '1'], ['4', '2'], ['4', '3']]
console.log(permutations(["1", "2", "3", "4"], 2));
// 6가지, [['1', '2'], ['1', '3'], ['1', '4'], ['2', '3'], ['2', '4'], ['3', '4']]
console.log(combinations(["1", "2", "3", "4"], 2));

// Case 5 (숫자로 이루어진 4개 원소 중 3개를 선택하는 경우)
// 24가지, [[1, 2, 3], [1, 2, 4], [1, 3, 2], [1, 3, 4], [1, 4, 2], [1, 4, 3], [2, 1, 3], [2, 1, 4], [2, 3, 1], [2, 3, 4], [2, 4, 1], [2, 4, 3], [3, 1, 2], [3, 1, 4], [3, 2, 1], [3, 2, 4], [3, 4, 1], [3, 4, 2], [4, 1, 2], [4, 1, 3], [4, 2, 1], [4, 2, 3], [4, 3, 1], [4, 3, 2]]
console.log(permutations([1, 2, 3, 4], 3));
// 6가지, [[1, 2, 3], [1, 2, 4], [1, 3, 4], [2, 3, 4]]
console.log(combinations([1, 2, 3, 4], 3));

    테스트 결과는 모두 예상대로 나왔다. 이제 순열과 조합의 이론과 구현이 끝났다.

    요약

    이번 포스트에서는 순열과 조합의 이론을 바탕으로 순열과 조합을 구현해보았다. 이를 통해 순열과 조합의 이해를 높이고, 이를 바탕으로 순열과 조합을 구하는 함수를 작성하였다. 간단하게 정리해보면 다음과 같다.

    1. 순열과 조합을 구현하기 위해 공식을 바탕으로 로직을 정리하였다.
      • Function Signature: input, output, function name을 정리
      • Pseudo Code: 함수의 동작을 단계별로 정리
    2. 정리한 로직을 함수 형태의 코드로 작성하고 3단계에 걸쳐 개선해보았다.
      • 단순 구현: for문을 활용하여 순열과 조합을 구현
      • 재귀 구현: 재귀를 활용하여 중첩 for문을 제거
      • 최적화: 로직을 개선하여 불필요한 동작을 제거
    3. 5가지 유형의 input을 통해 함수를 테스트하고, 순열과 조합을 구하는 것을 확인하였다.
      • 문자열, 숫자, 배열, 객체로 이루어진 input을 활용하여 테스트
