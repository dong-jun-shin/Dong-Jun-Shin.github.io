    블록체인을 이용한 식별과 DID 인증 방식

    2021, Nov 28

    블록체인이란

    • 블록체인은 수많은 노드(사용자)에게 하나의 데이터를 분산해 저장한다.
    • 새로운 노드(사용자)는 채굴을 통해 확보된다.
    • 기존 서비스는 중앙 서버에 데이터를 보관했다는 점에서 탈중앙화(Decentralized) 특징을 가지고 있다.

    블록체인 특성

    • 분산 네트워크를 사용해서 분산성
    • 오픈 소스로 활용성이 높아서 확장성
    • 모든 거래기록에 접근 가능해서 투명성
    • 모든 노드(사용자)가 공동으로 소유해서 무결성
    • 공동 소유로 인한 조작 방지가 가능해서 보안성

    블록체인 특징

    • 장점
      • 보안성 향상
        • 분산된 노드로 해킹이 불가능, 암호화된 데이터와 키 값으로만 거래됨
      • 비용 감소
        • 탈중앙화
      • 처리 속도 향상
        • 인증과 증명에 제 3자(발급처)가 필요하지 않음
      • 가시성 극대화
        • 가시성 극대화
          • 실시간으로 여러 노드에 대한 모니터링이 가능
    • 단점
      • 저장 공간 증가
        • 사용할수록 데이터가 늘어, 필요로 하는 저장 공간이 늘어남
      • 환경 오염
        • 채굴 과정에서 많은 양의 전기를 사용하여, 환경오염을 발생
      • 거래검증 주체
        • 전 세계에 분포된 노드가 거래검증의 주체이고, 익명의 검증인은 방대한 양의 컴퓨팅 파워를 이용해 거래를 증명해야 함
      • 처리 비용 낭비 (데이터 중복)
        • 참가한 모든 컴퓨터가 모든 자료를 다운 및 보관해야 함
      • 확장성의 한계
        • 오픈 소스로 활용성은 높으나, 시장에서 처리되는 대량의 데이터를 처리하는 것은 아직 불가능
      • 프라이버시 노출
        • 모든 사용자가 함께 거래내역을 처리하고 검증
      • 익명성의 한계
        • 법률적 문제 등에서 책임 소재가 불명확함

    DID란 (Decentralized Identifier, 탈중앙화 신원증명)

    • 개인이 자신의 정보에 완전한 통제권을 갖도록 하는 기술
    • DID를 활용한 인증 앱으로 인증하면, 다시 인증을 위한 인증을 할 필요가 없음

    DID의 인증 방식

    • 기존 인증 방식
      • 인증 방식에 필요한 구성원

        발급처, 사용자, 검증자 (ex. 질병관리청, 손님, 고기집 사장님)

      • 인증 과정

        발급처가 증명서를 보유
        > 검증자가 사용자에게 증명 요청
        > 사용자가 본인 인증을 통해 증명서 발급을 발급처에 요청
        > 사용자가 검증자에게 증명서를 전달
        > 검증자는 증명서의 입증을 위해 발급처에 전달해서 검증
        (필요한 증명서가 많을 수록, 각 발급처에서 이 과정을 거쳐야 함)

    • DID
      • 인증 방식에 필요한 구성원

        발급처, 사용자, 검증자 (ex. 질병관리청, 손님, 고기집 사장님)

      • 인증 과정

        발급처의 증명서를 사용자가 보유(Verifiable credentials),
        발급처는 증명서의 인증키를 블록체인 기반의 ‘검증정보 저장소(Verifiable Data Registry)’에 보관
        > 검증자가 사용자에게 증명 요청
        > 사용자는 필요한 증명서(VC)를 그룹으로 묶어서, 그룹(Verifiable Presentation)을 검증자에게 QR코드로 전달
        > 검증자는 QR코드에 VC 인증키들을 ‘검증정보 보관소’의 각 인증서 키와 비교해서 검증

    DID의 중요성

    • 공기업 뿐 아니라, 일반 기업에서도 인증정보를 활용한 서비스를 제공하는 것이 가능해짐
    • DID 인증 정보를 보관할 수 있는 ‘지갑 앱’을 통해, 개인의 정보들을 스마트폰에 저장하고 손쉽게 활용 가능해짐

      주체별 소유한 정보
      - 발급처(기관) - 증명서
      - 사용자(스마트폰 지갑 앱) - 증명서에 대한 인증키(VC)
      - 저장소 - 증명서에 대한 인증키(VC)
