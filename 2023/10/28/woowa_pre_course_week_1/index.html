    배경

    현재 2년차 개발자로 일하고 있었지만, 아직 부족함을 많이 느끼고 있었고 더 생산성 있는 코드 작성을 위해 많은 고민을 하고 있었다.

    그러던 중 우테코에서 6기를 모집한다는 글 중 프리코스는 누구나 참여할 수 있다는 내용을 접하게 되었다. 좀 더 자세히 읽어보니, 프리코스를 진행하는 것만으로도 많은 것을 배울 수 있을 것이라는 글과 함께 후기들이 좋아서 흥미를 가졌다.

    개인 시간을 활용해서 참여했던 1개월간의 배운 점과 후기들을 잊지 않도록 정리하고자, 이 글을 포함해서 4개의 포스트로 정리하고자 한다.

    과제 진행 셀프 QnA

    • Java8도 있는데, 왜 Java17이 과제 기술 스택으로 선택되었을까?
      • 타 버전 대비 긴 Support 기간
      • Java17에서 최신 버전으로 버전업할 경우 영향 최소화
      • 버전업 시, 스프링 등 관련 라이브러리, 프레임워크, 플랫폼의 호환성
    • Java JDK 17을 선택해야 하는데, OracleJDK와 OpenJDK가 있다고 한다. OracleJDK는 유료인데 OpenJDK를 사용해야 하는 것인가?
      • OracleJDK는 BCL(Binary Code License)를 적용하기 떄문에, 개인이 학습 또는 테스트 목적 등(일반적인 목적)의 용도로 쓰면 무료라고 한다. 비상업적인 용도로 사용하는 것에 한해 괜찮다고 한다.
      • 그러므로 Oracle JDK 17로 과제를 진행하기로 결정
    • Java JDK 설치 파일로 archive, msi, exe가 있다… 무엇을 선택해야 하지?
      • archive: 버전별 설치 파일의 보관을 목적으로 압축되어 있는 파일
      • msi: 윈도우가 제공하는 설치 파일 확장자로 PC에 파일 복사 및 레지스트리 값을 설정
        • Microsoft Silent Intaller의 약자
        • msiexec.exe라는 윈도우 인스톨러 엔진이 있고 요구 버전이 맞아야 실행 가능
        • 기업 대상 프로그램 배포 및 업데이트 할 때, 대상 PC 운영에 용이
          • e.g. MS의 Active Directory를 통해 msi 파일을 배포
      • exe: 독립적으로 실행할 수 있는 확장자로 프로그램을 실행하거나 설치
        • Executable file format의 약자
        • exe의 구성은 ‘msi + 실행에 필요한 환경 구성 파일’ 이라고 볼 수 있음
        • 설치 전 환경을 확인하고, 필요한 엔진, 프레임워크 등을 안내하거나 자동 설치
        • 프로그램 설치 시 사용할 언어를 선택하는 옵션 제공
      • 그러므로 범용적으로 사용되기도 하고, 설치되는 PC의 환경까지 확인해주는 exe를 다운받기로 결정
    • Java linting
      • continuation indent이 vscode에는 없다. 어떻게 설정하지?
        • 조사를 해보니, vscode에서는 ‘Wrapping Indent’라는 이름으로 존재하며, ‘none, same, indent, deepindent’ 옵션이 있다.
          • Wrapping Indent란 줄바꿈 시, 첫 줄 다음 줄부터 적용되는 indent를 의미
          • none: indent를 적용하지 않는다.
          • same: 상위의 indent와 똑같이 적용한다.
          • indent: indent를 1개 적용한다.
          • deepindent: indent를 2개 적용한다.
    • vscode에 google-java-format 적용하기 (in windows)
      • linting 적용을 위해, ‘ilkka.google-java-format’ extention 설치 후, 설치된 extention의 ‘extension.js’ 경로를 찾아서 설정
      • ‘Format Document’, ‘Format on save’ 선택

    배운 점

    • 요구사항을 정확히 준수함
      • 기능 요구사항, 프로그래밍 요구사항, 과제 진행 요구사항을 정확히 준수하여 프로젝트의 목표 달성과 코드 품질을 보장함.
      • 요구사항 준수로 인해 개발 과정에서의 혼란을 줄이고 효율적인 협업을 가능하게 함.
    • 커밋 메시지를 의미 있게 작성함
      • 각 커밋 메시지가 작업 내용을 명확히 설명하여 코드 변경 이력을 추적하고 이해하기 쉽게 함.
      • 의미 있는 커밋 메시지는 코드 리뷰와 협업 과정에서의 오류를 줄이고 효율적인 버전 관리를 지원함.
    • 버전 관리 시스템을 통해 관리할 자원에 대해서도 고려함
      • 불필요한 파일을 버전 관리 시스템으로 관리하지 않음으로써 저장소 크기를 줄이고 관리 비용을 절감함.
      • 클린한 저장소 관리로 코드 배포와 협업 프로세스를 최적화하여 개발 효율성을 높임.
    • Pull Request를 보내기 전 브랜치를 확인함
      • 기능 구현은 별도 브랜치에서 진행하고 PR을 통해 코드 충돌을 방지하며 코드 품질을 유지함.
      • PR을 통해 코드 변경 사항을 명확하게 검토하고 팀 내 협업을 강화하여 개발 생산성을 높임.
    • 이름을 통해 의도를 드러냄
      • 변수, 함수, 클래스 이름을 명확하게 지어 코드의 의도를 명확히 전달하고 가독성을 높임.
      • 의미 있는 이름 사용으로 코드 이해도를 높이고 오류 가능성을 줄여 개발 생산성을 향상시킴.
    • 축약하지 않음
      • 축약 없이 명확하고 자명한 이름을 사용하여 코드의 이해를 촉진하고 유지보수성을 높임.
      • 명시적인 이름은 오류를 줄이고 코드 리뷰와 협업 시간을 단축시켜 개발 효율성을 증대시킴.
    • 공백도 코딩 컨벤션에 맞춤
      • 제어문(if, for, while 등) 사이에 적절한 공백을 사용하여 코드의 가독성을 높이고 일관된 스타일을 유지함.
      • 일관된 공백 사용은 코드를 더 쉽게 이해하고 유지보수할 수 있게 도와줌.
    • 공백 라인을 의미 있게 사용함
      • 공백 라인을 사용하여 코드 블록을 시각적으로 구분하고 관련 있는 코드 그룹을 형성하여 가독성을 향상시킴.
      • 의미 있는 공백 사용은 코드의 구조를 명확히 하고 코드 논리를 시각적으로 분리하여 코드 변경을 쉽게 만듦.
    • space와 tab을 혼용하지 않음
      • 들여쓰기에 일관된 방식(space 또는 tab)을 사용하여 코드의 일관성을 유지하고 혼동을 방지함.
      • 일관된 들여쓰기 스타일은 코드의 시각적인 일관성을 유지하고 협업 시 오류를 줄여줌.
    • 의미 없는 주석을 달지 않음
      • 변수와 함수 이름을 명확하게 지어 코드 자체가 설명할 수 있도록 하여 주석을 최소화함.
      • 명확한 코드는 의도를 명확히 전달하고 코드 이해도를 높여 유지보수성을 개선함.
    • IDE의 코드 자동 정렬 기능을 활용함
      • IDE의 자동 정렬 기능을 활용하여 일관된 코드 스타일을 유지하고 코드 포맷팅 작업을 자동화함.
      • 자동 정렬은 시간 절약과 함께 코드의 일관성을 유지하며 개발자들 간의 스타일 차이를 최소화함.
    • 프로그래밍 언어에서 제공하는 API를 적극 활용함
      • 언어의 API를 적극 활용하여 이미 검증된 기능을 사용함으로써 코드의 안정성과 효율성을 높임.
      • API 사용은 개발 시간을 단축하고 코드의 일관성을 유지하며 유지보수 비용을 최소화하는 데 도움을 줌.
      // 예시: JavaScript에서 Array.prototype.join을 활용한 문자열 생성
let members = ["pobi", "jason"];
let result = members.join(","); // "pobi,jason"
console.log(result);

// 예시: JavaScript에서 Array.prototype.includes를 활용한 값 포함 여부 확인
let members = ["pobi", "jason"];
let isIncluded = members.includes("pobi"); // true
console.log(isIncluded);

    마무리

    첫 주는 워밍업으로 숫자 야구 구현을 하였다. 평소 일정에 치여, 정상 작동을 우선시하여 지나쳤던 부분들에 대해 하나씩 짚어가면서 참여해보았다. 간단한 숫자 야구 구현이었지만, 그 안에서 나의 빈약한 기본기를 느낄 수 있었고 하나씩 다시 되새기며 도움이 되는 것을 느낄 수 있었다.

    앞으로 3주동안 더 많은 걸 배울 수 있을 것이고, 항상 느끼던 빈약한 기본기가 채워지는 계기가 되지 않을까 하는 마음으로 1주차를 마무리한다.
